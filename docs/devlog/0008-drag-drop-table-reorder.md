# Drag-and-drop для изменения порядка персонажей в таблице

## 🎯 Проблема
В режиме редактирования персонажей в табличном представлении не было возможности изменять порядок отображения персонажей. Drag-and-drop работал только в режиме карточек.

## ✅ Решение
Добавлена кнопка drag handle (↕) в столбец "Действия" для каждого персонажа в таблице. Реализована логика drag-and-drop с валидацией: gold recipients могут перемещаться только внутри своей секции, non-gold персонажи — только внутри своей.

## 📝 Изменённые файлы
1. `src/components/organisms/CharacterTable.vue` — добавлены drag state, handlers, emit `reorder`, кнопка drag handle, стили для visual feedback
2. `src/components/organisms/CharacterList.vue` — добавлен handler `handleReorder` для обработки reorder emit от таблицы

## 🚀 Как протестировать
1. Переключиться в режим редактирования персонажей
2. Выбрать табличное представление
3. В столбце "Действия" появится кнопка ↕ рядом с кнопками редактирования/удаления
4. Перетащить персонажа за кнопку ↕ на другую позицию в той же секции
5. Персонаж из секции gold recipients нельзя перетащить в секцию "Остальные" (и наоборот)
6. При наведении на допустимую позицию — зелёная подсветка, на недопустимую — красная

## ⚙️ Важные детали
- Drag handle имеет `draggable="true"` только на кнопке (не на всей строке)
- Валидация через `canDropTo()` сравнивает `isGoldRecipient` у source и target персонажей
- Visual feedback: `.dragging` (opacity 0.5), `.drag-over` (primary-light), `.drag-invalid` (red)
- Emit передаёт глобальные индексы from/to в родительский CharacterList
- `CharacterList.handleReorder()` выполняет splice операцию и вызывает `charactersStore.reorderCharacters()`

## 🎨 Дизайн drag handle
Кнопка drag handle стилизована как заметный элемент интерфейса:
- Размер 28x28px с фиксированной шириной для консистентности
- Градиентный фон с эффектом "рельефности"
- Вертикальные полосы (⠿) внутри кнопки, символизирующие возможность перетаскивания
- Hover: подсветка primary цветом, scale 1.05, тень
- Active: scale 0.95 для тактильного feedback
- Focus-visible: outline для accessibility

## 🎉 Итог
Функциональность drag-and-drop теперь доступна в обоих режимах отображения: карточках и таблице. Кнопка drag handle заметная и интуитивно понятная.
